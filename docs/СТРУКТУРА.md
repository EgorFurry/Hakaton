# Структура проекта IncluBank

```
Hakaton/
│
├── .gitignore                      ← игнорируем мусор
├── README.md                       ← описание проекта
│
├── backend/                        ← Никита + Егор
│   ├── main.py                     ← запуск FastAPI
│   ├── requirements.txt            ← зависимости Python
│   ├── .env                        ← секреты (не пушится в git!)
│   ├── .gitignore                  ← локальный игнор для бека
│   │
│   ├── config/
│   │   ├── __init__.py
│   │   └── settings.py             ← порты, часы АБС, пути к файлам
│   │
│   ├── routes/
│   │   ├── __init__.py
│   │   ├── user.py                 ← GET /api/user/:id
│   │   ├── payment.py              ← /api/payment/create, /history, /process
│   │   ├── accessibility.py        ← /api/accessibility/settings, /ai/ask
│   │   └── admin.py                ← /admin/abs, /admin/sync, /admin/pending
│   │
│   ├── services/
│   │   ├── __init__.py
│   │   ├── abs_service.py          ← логика middleware, синхронизация АБС
│   │   ├── db_service.py           ← чтение/запись JSON файлов
│   │   └── ai_service.py           ← Gemini API интеграция
│   │
│   ├── data/                       ← JSON хранилище (не пушится в git!)
│   │   ├── users.json
│   │   ├── payments.json
│   │   ├── queue.json
│   │   └── accessibility.json
│   │
│   └── utils/
│       └── __init__.py
│
├── frontend/                       ← Бекболат
│   ├── index.html                  ← главный PWA файл
│   ├── manifest.json               ← PWA манифест
│   └── static/
│       ├── css/
│       │   └── style.css
│       └── js/
│           └── app.js
│
└── docs/                           ← документация
    ├── API_ДЛЯ_БЕКБОЛАТА.md        ← контракт API с примерами JS
    ├── СТРУКТУРА.md                 ← этот файл
    └── DEMO_SCRIPT.md              ← сценарий для защиты перед жюри
```

## Правила работы с ветками

- `main`     — стабильная версия, сюда пушим только рабочий код
- `backend`  — Никита и Егор работают здесь
- `frontend` — Бекболат работает здесь

## Когда что-то готово

```bash
# Сделал фичу — пушишь в свою ветку
git add .
git commit -m "feat: добавил синхронизацию АБС"
git push origin backend

# Когда всё готово к финалу — мержим в main
git checkout main
git merge backend
git push origin main
```

## Эндпоинты (кратко)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | /api/user/:id | Данные пользователя |
| GET | /api/credit/status/:id | Статус кредита |
| POST | /api/payment/create | Создать платёж |
| GET | /api/payment/process/:id | Статус платежа |
| GET | /api/payment/history/:id | История платежей |
| POST | /api/accessibility/settings | Сохранить режим доступности |
| POST | /api/accessibility/ai/ask | Вопрос AI-ассистенту |
| GET | /admin/abs-status | Статус АБС |
| POST | /admin/abs | Включить/выключить АБС (демо) |
| POST | /admin/sync | Синхронизировать очередь |
| GET | /admin/pending | Список платежей в очереди |
